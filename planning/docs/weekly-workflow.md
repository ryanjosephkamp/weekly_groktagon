# Weekly Workflow

## Purpose

This workflow supports the 4-week pilot described in the locked project plan. Each week is prepared under `weekly/YYYY-WXX/`, with component reports stored in `reports/` and the final public Markdown post stored as `final-blog-post.md`.

## Weekly Cycle

1. Daily source collection runs through the scheduled workflow.
2. Raw collection output is stored only as workflow artifacts.
3. The weekly folder is reviewed before report drafting begins.
4. Eight component reports are prepared for the target week.
5. The final Markdown post is assembled from the eight reports.
6. The final post is checked for citations, original wording, and public/private separation before publication.

## Week Folder Rules

- Use `weekly/YYYY-WXX/` for each weekly edition.
- Store report drafts in `weekly/YYYY-WXX/reports/`.
- Keep raw source collection output out of committed repository paths.
- Use `weekly/YYYY-WXX/final-blog-post.md` for the final Markdown post.
